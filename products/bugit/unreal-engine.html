How It Works in Unreal Engine

1

Add RVR Plugin to Your Project

Runtime Video Recorder handles the capture side. It runs in-engine, capturing video, inputs, and logs with minimal performance impact (1-3% CPU).

Trigger Bug Report (Blueprint or C++)

When a tester finds a bug, one function call packages everything into a bundle and uploads to BugIt.

// C++ - Trigger bug report
UBugItCaptureSubsystem* BugIt = GetGameInstance()->GetSubsystem<UBugItCaptureSubsystem>();
BugIt->ReportBug(TEXT("Player fell through floor"));

Also available as Blueprint nodes - no C++ required.

3

View in BugIt Dashboard

Developers open the self-hosted BugIt dashboard. Video, inputs, logs, and frame timing are all synced. Scrub to any moment and see exactly what happened.

Frequently Asked Questions

How do I report bugs with video in Unreal Engine?

BugIt + RVR captures video, inputs, logs, and frame timings directly from your UE game. When a bug occurs, press one button to save everything as a bundle. Open the BugIt dashboard to see all data synced together.

Does BugIt work with UE5?

Yes, BugIt works with Unreal Engine 4.25+ and all UE5 versions including 5.4. The capture is powered by Runtime Video Recorder (RVR), which fully supports UE5.

Is BugIt free for Unreal Engine projects?

Yes, BugIt (the dashboard) is open source and free forever under the MIT license. You need RVR (sold separately) for the capture side - it's the plugin that runs in your UE project.

What's the performance impact?

RVR uses hardware-accelerated encoding with 1-3% CPU overhead. Recording runs on a separate thread and doesn't block the game thread, so your testing environment stays accurate.
